事件委托

起因：动态添加的li标签，无法完成click事件，此时考虑事件委托

on(绑定事件，选择器，匿名函数)

**原理：**利用冒泡原理，在追加的元素的父级对象上，绑定on方法，在on方法中按格式（'绑定事件'，'选择器'，'匿名函数'），绑定事件，触发事件

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta http-equiv="X-UA-Compatible" content="ie=edge">
    <title>Document</title>
</head>
<body>
    <ul>
        <li>alex</li>
        <li>wxx</li>
    </ul>
    <button>添加</button>
    <script type="text/javascript" src="./js/jquery-3.3.1.js"></script>
    <script type="text/javascript">
        // $('li').click(function(){
        //     alert($(this).text());
        // });

        // 事件委托的on方法，格式为（'绑定事件'，'选择器','匿名函数'），原理：利用冒泡原理，把事件追加到父级上，触发执行效果
        $('ul').on('click','li',function(){
            // 当前的$(this)表示on事件中，li的jquery对象，输出他的文本内容
            alert($(this).text());
        });

        // 点击添加按钮，添加的li标签，是不能触发li标签的点击事件，所以需要利用事件委托，将事件委托到父级触发
        $('button').click(function(){
            $('ul').append('<li>egon</li>');
        });
    </script>
</body>
</html>
```
